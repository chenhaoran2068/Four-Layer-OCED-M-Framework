# Repository Scope

This repository is intentionally a framework repository.

## Allowed Content

The repository should contain:

- framework-level documentation
- public-facing terminology and definitions
- minimal schemas
- minimal templates
- worked examples
- distilled instantiation profiles rewritten into public-safe, relative-path
  examples
- synthetic trace, gate, registry, or handoff artifacts that explain the
  framework
- GitHub-safe helper scripts
- figures or diagram notes that explain the framework

## Excluded Content

The repository should not become a dumping ground for:

- local private workspaces
- active manuscript build clutter
- internal memory exports
- private project notes
- workstation-specific absolute paths
- sensitive data or private derived data
- large local production outputs
- internal-only operational scripts
- raw mirrors of active local system roots
- historical dry-run bundles or approval logs copied directly from local
  operations

## Practical Rule

If a file mainly answers "what is the framework and how does it work?", it may
belong here.

If a file mainly answers "what did we do on one machine, in one private project,
with one local production surface?", it probably belongs elsewhere.
