# Public Release Messaging

## Purpose

This file keeps the public release wording aligned across:

- the framework repository homepage
- GitHub repository description text
- companion-repository references
- public announcement copy

## Canonical Short Positioning

Use the following short positioning consistently:

- `Four-Layer-OCED-M-Framework` explains the framework
- `Research_Workflow` is the main public workflow-facing companion repository
  for the current research workflow pack
- `Research_Workflow` presents a cleaned public workflow example with explicit
  reading, validation, agent-entry, and frozen-evidence surfaces
- neither repository is a full public runtime mirror

## Recommended GitHub About Text

### Framework Repository

Recommended text:

`Framework for governable agentic workflows: Four-Layer + OCED-M, linked to the Research_Workflow public workflow pack.`

### Workflow Repository

Recommended text:

`Public workflow pack for the Research workflow: a cleaned workflow-facing companion repository to Four-Layer-OCED-M-Framework.`

## Recommended Homepage Lead

Recommended short lead for the framework homepage:

- the framework repository explains the framework
- the workflow repository carries the main public workflow pack
- neither repository should be described as a full runtime mirror

## Announcement Rule

When both repositories are already public:

- framework-first announcements should mention `Research_Workflow` by name
- if only one link is used in the first short post, the companion repository
  should be linked in a follow-up post or thread step
- do not describe the framework repository as if it already contains the full
  workflow pack

## Claim-Safety Rule

Safe wording:

- framework/specification repository
- workflow-facing companion repository
- main public workflow instantiation repository
- cleaned public workflow example
- not a full public runtime mirror

Avoid wording:

- complete public runtime
- full local system release
- everything is already mirrored publicly
