# Condensed Stage Map

This condensed map is a public-safe summary of a governed research workflow.

1. Intake and project formation
   - convert a natural-language question into a governed project shell
2. Source inventory and seed evidence review
   - check whether the required data, literature, and reporting assets exist
3. Protocol lock
   - freeze the main question, main analysis boundary, and redesign rules
4. Knowledge build
   - deepen the literature subset and novelty understanding
5. Data build
   - create a rebuildable analysis dataset with lineage
6. Analysis execution
   - run primary, validation, and sensitivity analyses under controlled revision
7. Publishability and value review
   - decide whether the branch is worth writing up
8. Professional review panel
   - stress-test the branch before manuscript drafting
9. Writing
   - convert evidence and results into a controlled manuscript package
10. Citation control
    - verify that citations are real, relevant, and traceable
11. Journal fit
    - align the package to a target journal
12. Submission packaging
    - assemble the final submission-ready bundle
13. Three-round QA
    - check consistency, adequacy, and package integrity
14. User approval and archive
    - route the final package to explicit human approval and freeze the archive

The example coordinate `r_protocol_lock` governs the boundary between Stage 2
and Stage 3.
